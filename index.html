This plugin takes a sequence of image slices stored in a folder, and delivers a list of registered image slices (with enlarged canvas). One of the images in the sequence can be selected by the user as reference and it will remain intact.

The plugin can perform 6 types of image registration techniques:

  • Translation
  • Rigid (translation + rotation)
  • Similarity (translation + rotation + isotropic scaling)
  • Affine
  • Elastic (via bUnwarpJ with cubic B-splines)
  • Moving least squares

All models are aided by automatically extracted SIFT features.

Install

ubuntu: $ apt-get install imagej-register-virtual-stack-slices
osx: $ brew install imagej-register-virtual-stack-slices
windows: $ choco install imagej-register-virtual-stack-slices

Configure

  # Filename: register_virtual_stack_slices.launch

  package: lost_comms_recovery
    node: register_virtual_stack_slices

      subscribe:
        stack: stack
        sift_features: sift_features

      publish:
        stack_registered: stack_registered
        transformations: transformations

      params:
        registration_model: rigid
        reference_image_index: 1

Run

$ fiji-launch register_virtual_stack_slices register_virtual_stack_slices.launch

Node API

Subscribed Topics

stack (stack_msgs/stack_xyz)
  • The image stack that needs to be aligned.
sift_features (feature_msgs/sift)
  • SIFT features of the image stack used for image alignment. For example published by the Feature_Extraction node.

Published Topics

stack_registered (stack_msgs/stack_xyz)
  • The resulting aligned image stack.
transformations (geometry_msgs/tranform)
  • The transformations performed on each image.
relaxation_graph (std_msgs/xy_vector)
  • For constrain_shrinkage only. Used to visualize the relaxation of the system.
status (std_msgs/status)
  • Indication of how much work is remaining.
log (std_msgs/log)
  • Log messages.

Parameters

~registration_model (string, default: "rigid")
  • The image registration technique to be used. Options are:
    • Translation
    • Rigid (translation + rotation)
    • Similarity (translation + rotation + isotropic scaling)
    • Affine
    • Elastic (via bUnwarpJ with cubic B-splines)
    • Moving least squares
~reference_image_index (int, default: 1)
  • The image registration technique to be used. Options are:
~constrain_shrinkage (bool, default: "false")
  • Option to constrain the registration to avoid section shrinking for large sets of images. If this option is used, no reference image needs to be selected. All images will be transformed into the same common space by initializing first the system with a rigid transformation and then relaxing it based on the desired registration model.
~shear_regularization (int, default: 0.95)
  • For constrain_shrinkage only. The shear.
~scaling_regularization (int, default: 0.95)
  • For constrain_shrinkage only. The scaling.
~isotropy_regularization (int, default: 0.95)
  • For constrain_shrinkage only. The isotropy or aspect ratio.
~elastic_registration_mode (string, default: "Mono")
  • For registration_model="elastic" only. Choose registration mode "Mono", "Accurate" or "Fast". "Mono" means unidirectional registration and its actually the fastest of the three modes. "Accurate" and "Fast" perform bidirectional registration and take more time.
~elastic_subsampling_factor (int, default: 0)
  • For registration_model="elastic" only. The image scaling factor from 2 (1) to 2 (128). We recommend to use it if the images are very large.
~elastic_initial_deformation (string, default: "Very Coarse")
  • For registration_model="elastic" only. "Very Coarse" to "Super Fine". They define the number of B-spline coefficients in the deformation grid (from 2x2 to 16x16). More coefficients mean more precision but also the possibility of over-registering.
~elastic_final_deformation (string, default: "Fine")
  • For registration_model="elastic" only. "Very Coarse" to "Super Fine". They define the number of B-spline coefficients in the deformation grid (from 2x2 to 16x16). More coefficients mean more precision but also the possibility of over-registering.
~elastic_divergence_weight (float, default: 0)
  • For registration_model="elastic" only. Regularize the deformation to make it smooth.
~elastic_curl_weight (float, default: 0)
  • For registration_model="elastic" only. Regularize the deformation to make it smooth.
~elastic_landmark_weight (float, default: 0.0)
  • For registration_model="elastic" only. Relevance of adjusting the correspondences found in the feature extraction.
~elastic_image_weight (float, default: 1.0)
  • For registration_model="elastic" only. Relevance of the similarity between source and target image in the energy function.
~elastic_consistency_weight (float, default: 10.0)
  • For registration_model="elastic" only. Relevance of the consistency error between the direct and inverse deformations (only for "Accurate" or "Fast" modes).
~elastic_stop_threshold (float, default: 0.01)
  • For registration_model="elastic" only. The difference between the last and previous algorithm iterations that makes the registration to end.

More Details

  1. The plugin is multithreaded: more CPU cores means faster execution.
  2. The two most relevant parameters in the SIFT feature extraction are:
    • maximum image size, which limits the low-end size of the features (i.e. decrease maximum image size to increase the size of the smallest features to extract.)
    • inlier ratio, which determines when to reject the intersection set of feature correspondences between two images (ratio between correspondent features vs. all extracted features.)
  3. All types of images are accepted (8-bit, 16-bit, 32-bit and RGB) but only in .tif, .jpg, .png, .gif, .tiff, .jpeg, .bmp, .pgm, or .ima format. Output images are saved as TIFF.
